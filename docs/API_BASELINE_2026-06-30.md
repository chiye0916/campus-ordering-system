### 支付与订单状态边界

- 支付订单成功，返回 `data.status=2`
- 支付流水表存在 `MOCK` 成功记录
- 已支付订单不能直接取消，返回 `409`
- 管理员完成订单成功
- 已完成订单不能取消，返回 `409`
- 已完成订单不能重复支付，返回 `409`
- 空购物车不能下单，返回 `409`
- 请求体 JSON 格式错误，返回 `400`

### Redis 订单状态锁

并发支付测试订单：

```text
LOCK_ORDER_ID=12
```

验证结果：

```text
两个并发支付请求：一个返回 200，一个返回 409
payment_record 中该订单只有 1 条 MOCK 成功支付流水
Redis key lock:order:status:12 已释放，exists=0
```

说明当前 Redis 分布式锁与数据库状态条件更新共同生效：

- Redis 锁用于减少同一订单并发状态流转进入业务逻辑；
- 数据库 `where id = ? and status = ?` 条件更新作为最终一致性保护；
- 支付流水不会因为并发支付产生多条成功记录。

当前订单状态：


当前合法流转：

```text

```

当前取消规则：

```text
只有待支付订单可以直接取消。
已支付订单不能直接取消，后续如果需要取消已支付订单，应单独设计退款流程。
```
